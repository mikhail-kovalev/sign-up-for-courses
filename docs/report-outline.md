# План отчета по курсовой работе

## 1. Введение

Тема работы: разработка клиент-серверного мобильного приложения записи на онлайн курсы образовательной организации.

Цель работы: разработать Android-приложение, которое позволяет пользователю авторизоваться, просматривать каталог онлайн-курсов и записываться на выбранный курс через серверное API.

## 2. Анализ предметной области

Образовательная организация публикует список онлайн-курсов. Пользователь проходит регистрацию или авторизацию, просматривает доступные курсы, открывает карточку курса и оформляет запись.

Основные сущности:

- пользователь;
- курс;
- запись пользователя на курс.

## 3. Функциональные требования

- регистрация пользователя;
- вход пользователя;
- просмотр списка курсов;
- просмотр подробной информации о курсе;
- запись на курс;
- просмотр своих записей.

## 4. Нефункциональные требования

- клиент-серверная архитектура;
- back-end на Ktor;
- локальный сервер БД PostgreSQL;
- Android-клиент на Jetpack Compose;
- архитектура Clean Architecture;
- хранение бизнес-данных не в Firebase/Firestore.

## 5. Архитектура приложения

Клиентская часть разделена на слои:

- `data` - работа с сетью, DTO, хранение токена, реализации репозиториев;
- `domain` - модели предметной области, интерфейсы репозиториев, use cases;
- `presentation` - экраны Jetpack Compose, ViewModel, навигация.

Серверная часть содержит слои:

- `domain` - модели и контракты;
- `application` - сценарии использования;
- `data` - PostgreSQL, JDBC-репозитории, миграции;
- `presentation` - REST DTO;
- `security` - JWT и хеширование паролей.

## 6. Проектирование базы данных

База данных содержит таблицы:

- `users` - пользователи;
- `courses` - онлайн-курсы;
- `enrollments` - записи пользователей на курсы.

Связь `users` и `courses` реализована через таблицу `enrollments`.

## 7. Реализация серверной части

Back-end реализован на Ktor. Для авторизации используется JWT. Пароли сохраняются в виде BCrypt-хеша. При старте сервер выполняет создание таблиц и заполняет базу демонстрационными курсами.

## 8. Реализация клиентской части

Android-приложение реализовано на Kotlin и Jetpack Compose. Для сетевого взаимодействия используется Ktor Client. Токен авторизации сохраняется локально в `SharedPreferences`.

## 9. Тестирование

Проверяются сценарии:

- вход тестового пользователя;
- регистрация нового пользователя;
- загрузка каталога курсов;
- открытие карточки курса;
- запись на курс;
- отображение записи в профиле.

## 10. Заключение

В результате работы разработано клиент-серверное Android-приложение для записи на онлайн-курсы образовательной организации. Приложение соответствует требованиям: использует Ktor back-end, локальную PostgreSQL БД, Jetpack Compose и разделение на слои Clean Architecture.
